Canonical vs Mutable 模式

这是 STS2 模型系统的核心设计：

  • 每个模型有一个 Canonical（只读原型） 实例，存储在 ModelDb 中
  • 运行时使用 ToMutable() 创建可变副本
  • 通过 AssertMutable() / AssertCanonical() 保护数据一致性
  • ModelId 由 Category + Entry 两部分组成，用于标识每个模型
// 获取 canonical 实例
CardModel canonical = ModelDb.Card<Bash>();

// 创建可变副本
CardModel mutable = canonical.ToMutable();
mutable.AssertMutable(); // OK

ModelDb — 模型数据库

ModelDb 是静态类，管理所有模型的注册和查询：

ModelDb.Card<Bash>()           // 获取卡牌
ModelDb.Power<VulnerablePower>()  // 获取力量
ModelDb.Monster<Cultist>()     // 获取怪物
ModelDb.AllCards                // 所有卡牌
ModelDb.AllRelics              // 所有遗物
ModelDb.AllPowers              // 所有力量
ModelDb.AllPotions             // 所有药水

模型类型通过 [GenerateSubtypes] 源码生成器自动注册，同时支持反射加载 Mod 中的子类型。

7. Hooks/ (2 文件) — Hook 事件系统

Hook.cs 是一个巨大的静态类，提供所有游戏事件钩子。

工作原理

每个钩子遍历 CombatState.IterateHookListeners() 返回的所有 AbstractModel（包括遗物、力量、怪物等），调用对应的虚方法：

// Hook.cs 中的典型模式
public static async Task BeforeAttack(CombatState combatState, AttackCommand command)
{
    foreach (AbstractModel model in combatState.IterateHookListeners())
    {
        await model.BeforeAttack(command);
        model.InvokeExecutionFinished();
    }
}

二、核心架构模式

Model-Entity-Node 三层架构

Models/   → 数据定义层（CardModel, RelicModel...）  — 定义"是什么"
Entities/ → 运行时实体层（Creature, CardPile...）   — 定义"怎么运作"
Nodes/    → 表现层 Godot Node（NCard, NCombatRoom...）— 定义"怎么显示"
  • Models 定义游戏对象的属性和行为逻辑
  • Entities 管理运行时状态和枚举类型
  • Nodes 处理 Godot 场景树中的视觉表现

数值系统

  • 所有游戏数值使用 decimal 类型（非 int 或 float），确保精度
  • DynamicVar 系统处理数值定义、修改和 UI 显示
  • DynamicVarSet 是每个模型拥有的动态变量集合
  • ValueProp 标记数值属性类别（用于 Hook 修改判断）

常见 DynamicVar 子类型：

DamageVar(8m, ValueProp.Move)      // 伤害变量
BlockVar(5m)                        // 格挡变量
PowerVar<VulnerablePower>(2m)       // 力量层数变量
MagicVar(3m)                        // 通用数值变量
EnergyVar(1m)                       // 能量变量

异步编程模式

STS2 大量使用 async/await，与 Godot 帧循环结合：

// 等待 Godot 帧信号
await Engine.GetMainLoop().ToSignal(Engine.GetMainLoop(), SceneTree.SignalName.ProcessFrame);

四、卡牌开发完整模板

using System.Collections.Generic;
using System.Threading.Tasks;
using MegaCrit.Sts2.Core.Commands;
using MegaCrit.Sts2.Core.Entities.Cards;
using MegaCrit.Sts2.Core.GameActions.Multiplayer;
using MegaCrit.Sts2.Core.HoverTips;
using MegaCrit.Sts2.Core.Localization.DynamicVars;
using MegaCrit.Sts2.Core.Models.Powers;
using MegaCrit.Sts2.Core.ValueProps;

namespace MegaCrit.Sts2.Core.Models.Cards;

public sealed class MyCard : CardModel
{
    // 悬浮提示（显示关联的力量/关键词说明）
    protected override IEnumerable<IHoverTip> ExtraHoverTips =>
        new[] { HoverTipFactory.FromPower<VulnerablePower>() };

    // 动态变量定义（用于描述文本中 {damage} {vulnerable} 的插值）
    protected override IEnumerable<DynamicVar> CanonicalVars => new DynamicVar[]
    {
        new DamageVar(8m, ValueProp.Move),
        new PowerVar<VulnerablePower>(2m)
    };

    // 构造函数：能量费用, 卡牌类型, 稀有度, 目标类型
    public MyCard()
        : base(2, CardType.Attack, CardRarity.Common, TargetType.AnyEnemy)
    {
    }

    // 打出效果
    protected override async Task OnPlay(PlayerChoiceContext choiceContext, CardPlay cardPlay)
    {
        // 造成伤害
        await DamageCmd.Attack(base.DynamicVars.Damage.BaseValue)
            .FromCard(this)
            .Targeting(cardPlay.Target)
            .WithHitFx("vfx/vfx_attack_blunt", null, "blunt_attack.mp3")
            .Execute(choiceContext);

        // 施加易伤
        await PowerCmd.Apply<VulnerablePower>(
            cardPlay.Target,
            base.DynamicVars.Vulnerable.BaseValue,
            base.Owner.Creature,
            this);
    }

    // 升级效果
    protected override void OnUpgrade()
    {
        base.DynamicVars.Damage.UpgradeValueBy(2m);
        base.DynamicVars.Vulnerable.UpgradeValueBy(1m);
    }
}
